# What is an Accessible Digital Textbook?

An **Accessible Digital Textbook (ADT)** is a digital version of a print textbook that has been structured, tagged, and enriched so that every learner — including those who use assistive technologies — can read, navigate, and interact with the content.

---

## Why Accessible Digital Textbooks Matter

Hundreds of millions of children around the world use physical textbooks that are unavailable in any accessible format. Learners who are blind, have low vision, have reading difficulties such as dyslexia, or rely on screen readers are often excluded from the same educational content as their peers.

An ADT closes that gap. It makes the same textbook content available:

- **To learners who are blind or have low vision** — via screen-reader-compatible HTML with proper heading structure, image alt-text, and ARIA landmarks
- **To learners with reading difficulties** — via read-aloud audio and clean, structured HTML that works with text-to-speech tools
- **To learners in low-connectivity environments** — via a self-contained offline web bundle that requires no internet connection after download
- **To learners in multilingual classrooms** — via built-in multi-language translation of all text content
- **On any device** — via responsive HTML that works in any browser without installing an app

---

## What an ADT Contains

A finished ADT produced by ADT Studio is a packaged web application — a folder of static HTML, CSS, JavaScript, images, and audio files — that includes:

| Component | Description |
|-----------|-------------|
| **Storyboard pages** | Each page of the textbook rendered as interactive HTML with proper semantic markup |
| **Quizzes** | Auto-generated multiple-choice, true/false, fill-in-the-blank, and other activities derived from the source content |
| **Glossary** | Key terms extracted and defined from the book |
| **Image captions** | Alt-text descriptions generated for every meaningful image |
| **Multilingual text** | The full text of the book translated into one or more target languages |
| **Text-to-speech audio** | Audio narration for every section, playable offline |

## Accessibility Standards

ADTs produced by ADT Studio are designed to conform to:

- **WCAG 2.1 Level AA** — the internationally recognised web content accessibility standard, covering perceivability, operability, understandability, and robustness
- **Semantic HTML5** — headings, landmarks, lists, tables, and figures are marked up using native HTML elements so that assistive technologies understand the document structure
- **ARIA** — where native HTML semantics are insufficient, ARIA roles and labels are used to convey meaning to screen readers

### Key accessibility features in the generated output

- **Logical heading hierarchy** — every page uses `<h1>`–`<h6>` in a meaningful order that screen-reader users can navigate
- **Alt text on all meaningful images** — decorative images are hidden from assistive technology; meaningful images carry descriptive text generated by the LLM
- **Skip navigation links** — users can jump directly to the main content region
- **Keyboard navigability** — all interactive elements (quiz answers, glossary entries) are reachable and operable via keyboard alone
- **Sufficient colour contrast** — text and interactive elements meet WCAG AA contrast ratios
- **Audio alternative** — every section has a text-to-speech audio track for learners who prefer listening

## ADTs vs. PDFs

| | PDF | ADT |
|--|-----|-----|
| Screen-reader friendly | ❌ Rarely (requires manual tagging) | ✅ Yes, by design |
| Works offline | ✅ Yes | ✅ Yes (packaged bundle) |
| Interactive quizzes | ❌ No | ✅ Yes |
| Multilingual | ❌ No (print content only) | ✅ Yes |
| Read-aloud audio | ❌ No | ✅ Yes |
| Works on any browser | ⚠️ Requires PDF viewer | ✅ Yes |
| Reflowable on mobile | ❌ No | ✅ Yes |
| Open format | ⚠️ Partially | ✅ Plain HTML/CSS/JS |

---

## Who Is an ADT For?

ADTs are designed to serve:

- **Learners with visual impairments** — the HTML output is navigable by any screen reader (NVDA, JAWS, VoiceOver, TalkBack)
- **Learners with dyslexia or print disabilities** — clean semantic HTML works well with read-aloud browser extensions and dedicated reading tools
- **Learners in low-resource settings** — the offline bundle works in any browser after a one-time download, with no internet required during use
- **Learners in multilingual education systems** — translated text is embedded directly in the bundle; learners can switch languages in the reader
- **Teachers and curriculum designers** — the ADT mirrors the structure of the print textbook, so the same lesson plans apply
